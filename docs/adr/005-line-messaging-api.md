# ADR-005: LINE Messaging API 統合

## ステータス

採用済み

## コンテキスト

ユーザーへの通知・レポート配信手段として、日本のユーザーに最もリーチしやすいチャネルを選定する必要があった。

## 決定

**LINE Messaging API** を統合し、以下の機能を提供する:

- 週次/月次スタッツレポートの自動配信
- アカウント連携（DARTSLIVE アカウントとの紐付け通知）
- 練習リマインダー

## 設計

### アカウント連携フロー

1. ユーザーが LINE 公式アカウントを友だち追加
2. LINE Webhook (`/api/line/webhook`) でフォローイベントを受信
3. リッチメニューからアカウント連携URLを発行
4. ユーザーが Web 上で LINE アカウントと darts Lab アカウントを紐付け
5. Firestore に `lineUserId` を保存

### レポート配信

- Cron ジョブ (`/api/cron/daily-stats`) が定期実行
- LINE 連携済みユーザーに Flex Message でレポート送信
- 週次: 今週のプレイ日数、平均 Rating、アワード集計
- 月次: 月間サマリー + 前月比較

### ステートマシン

- LINE Bot の対話は状態遷移ベースで管理
- 状態: `idle` → `awaiting_credentials` → `linking` → `linked`
- 不正な入力に対するグレースフルなエラーハンドリング

## 代替案

| 選択肢      | 不採用理由                                 |
| ----------- | ------------------------------------------ |
| メール通知  | 開封率が低い、プッシュ通知体験として劣る   |
| Web Push    | ブラウザ間の互換性、ユーザーの許可率の低さ |
| Discord Bot | 日本のダーツプレイヤー層との親和性が低い   |

## 結果

- 4 つの API ルート（webhook, link, credentials, status）で LINE 統合を実現
- Flex Message による視覚的なレポートで高いエンゲージメント
- ステートマシンパターンにより、複雑な対話フローを安全に管理
