# ADR-001: Firestore をメインDBに採用

## ステータス

採用済み

## コンテキスト

ダーツプレイヤー向けWebアプリとして、ユーザーのセッティング、スタッツ、記事、バレルデータなど多様なデータを永続化する必要がある。DB選定にあたり、以下の要件を考慮した:

- リアルタイム同期（練習中のデータ更新）
- サーバーレス環境（Vercel）での運用
- 認証との統合（Firebase Auth）
- コスト効率（個人開発規模）

## 決定

**Cloud Firestore** をメインデータベースとして採用する。

## 理由

1. **Firebase エコシステムとの統合**: Firebase Auth との認証統合がシームレス。セキュリティルールでユーザーごとのアクセス制御が宣言的に記述可能
2. **リアルタイム同期**: `onSnapshot` によるリアルタイムリスナーで、フォーカスポイントの即座の同期など、UXに直結する機能を低コストで実現
3. **サーバーレス親和性**: コネクションプール管理不要。Vercel Serverless Functions から直接アクセス可能
4. **スキーマ柔軟性**: ダーツスタッツのように項目が増減するデータに対して、NoSQL のスキーマレスな特性が有利
5. **コスト**: 無料枠が個人開発規模で十分（1日50K読み取り、20K書き込み）

## 代替案

| 選択肢                            | 不採用理由                                                             |
| --------------------------------- | ---------------------------------------------------------------------- |
| PostgreSQL (Supabase/PlanetScale) | コネクション管理の複雑さ、リアルタイム同期の追加実装コスト             |
| MongoDB Atlas                     | Firebase Auth との統合が別途必要、サーバーレス環境でのコールドスタート |
| DynamoDB                          | AWSエコシステムへの依存、Firebase Auth との分離                        |

## 結果

- 20以上のコレクションを運用中（users, darts, barrels, dartsLiveStats 等）
- サブコレクション活用（`users/{id}/dartsLiveStats`, `users/{id}/goals`）で論理的なデータ分離を実現
- Admin SDK によるサーバーサイドでの一括操作も問題なく動作
- リアルタイムリスナーは FocusPointsCard 等で活用中
