## 背景・課題

要件定義書（`docs/01-requirements.md`）の「1. 背景・目的」セクションが、課題の列挙と目的の記述のみにとどまっていた。既存サービスとの差別化ポイントや Darts Lab 独自の機能が明文化されておらず、以下の問題があった。

- 競合サービスとの比較が欠如しており、なぜ Darts Lab を作る必要があるのかが不明確
- DARTSLIVE / PHOENIX のデータ分断という重要な課題が記載されていなかった
- 完全オリジナル機能（LINE レポート配信、ヘルスケア連携、セッティング定量比較など）が要件として明記されていなかった
- 実装方針に関する注記（非公開 API の扱い、記事機能の方針など）が欠如していた

## 採用した方針

「1. 背景・目的」セクションを以下の 5 サブセクション構成に拡充した。

1. **1.1 背景**: 既存の課題リストに「マシンごとのデータ分断」を追加し、各課題の説明をより具体的に（0.1mm/0.1g 単位の定量性、禁煙/分煙フィルター、路線検索など）
2. **1.2 目的**: 既存内容を維持（微修正のみ）
3. **1.3 競合分析・差別化**: バレル検索・マイショップ・スタッツ分析の 3 カテゴリで既存サービスとの比較表を追加
4. **1.4 完全オリジナル機能**: LINE レポート配信、目標トラッキング、ヘルスケア連携、セッティング履歴・定量比較、高度な分析機能の 6 項目を表形式で整理
5. **1.5 実装方針の注記**: ディスカッション/記事の方針、非公開 API の扱い、DARTSLIVE/PHOENIX 連携の価値を明記

## 却下した選択肢

| 選択肢                               | 却下理由                                                                               |
| ------------------------------------ | -------------------------------------------------------------------------------------- |
| 別ドキュメントに競合分析を分離       | 要件定義書の「目的」と直結する内容のため、同一ドキュメント内に記載する方が参照しやすい |
| 技術詳細（非公開 API）を設計書に記載 | セキュリティ上の観点から設計書上では非公開とする方針を採用                             |
